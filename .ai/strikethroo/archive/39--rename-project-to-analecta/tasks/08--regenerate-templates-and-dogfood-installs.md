# Regenerate templates/ and dogfood harness installs

## Objective
Regenerate the generated trees from the edited source of truth: rebuild
`templates/` via the build pipeline and regenerate this repo's dogfooded
harness installs (`.claude/`, `.opencode/`, `.cursor/`, `.codex/`) by re-running
`init`, then confirm health with `doctor`.

## Skills Required
- **build-tooling**: run `npm run build` / `scripts/build-templates.mjs` and verify regenerated output.
- **cli**: run the built CLI `init`/`doctor` against this repo to refresh dogfood installs.

## Acceptance Criteria
- [ ] `npm run build` exits zero and regenerates `templates/` with `ana-*` skills/hooks and no `kb-*` artifacts.
- [ ] This repo's dogfood harness configs (`.claude/`, `.opencode/`, `.cursor/`, `.codex/`) are regenerated and reference `ana-*` skills/hooks, `ana-hooks/` (OpenCode), and updated hook command paths.
- [ ] `node dist/cli.js doctor` reports healthy.
- [ ] `grep -rIl -e 'kb-' -e 'knowledge-base' templates` returns no generated artifacts carrying the old brand (only intentional residue, if any).

Use your internal Todo tool to track these and keep on track.

## Technical Requirements
`templates/` is generated by `scripts/build-templates.mjs` and must never be
hand-edited. The dogfood `.claude/.opencode/.cursor/.codex` trees are `init`
output — stale hook wiring there breaks until regenerated. Regenerate, do not
hand-edit. Commit the regenerated artifacts.

## Input Dependencies
- Task 2 (renamed skills, CLI name, LauncherSkill type).
- Task 3 (renamed hook source files + wiring).
- Task 4 (`.ai/analecta` runtime paths).
- Task 5 (env vars, `.anaignore`, `ana-hooks/`, status label).

## Output Artifacts
- Regenerated `templates/` tree.
- Regenerated dogfood harness install directories.
- A built `dist/cli.js` (consumed by the dogfood KB migration and verification tasks).
